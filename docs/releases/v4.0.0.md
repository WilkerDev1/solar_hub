# Release v4.0.0 - Conectividad CRM, Gestión RBAC Dinámica y Motor de Tareas

**Fecha de Lanzamiento:** 2026-06-10  
**Estado:** Completado y Verificado  

---

## Resumen del Lanzamiento
La versión v4.0.0 añade la capa final de interacción relacional y seguridad avanzada para el Hito 4. Se introduce un motor relacional de tareas globales (`global_tasks`), la gestión de plantillas de rol en caliente (`role_permissions_templates`) con triggers reactivos en PostgreSQL, autenticación y autorización segura en endpoints administrativos vía tokens Bearer, y una experiencia de usuario altamente pulida con Combobox inteligente para vinculación de clientes con creación inline integrada, autocompletado de ocupaciones sugeridas, y la nueva interfaz de tareas personales con accesos directos optimizados para dispositivos móviles.

---

## Nuevas Características

### 1. Motor de Tareas Relacionales (`global_tasks`)
- **Tareas de Tipo Check y Entregable:** Soporte para tareas generales (checklist de confirmación directa) y tareas del tipo entregable (que requieren subir evidencia en el proyecto).
- **Asociación Relacional:** Las tareas se vinculan a la organización (`company_id`), opcionalmente a una obra/proyecto (`project_id`), se clasifican por su origen (`proyecto`, `administracion`, `consulta`, `almacen`), y se asignan a un perfil específico (`assigned_to`).
- **Vista de Tareas Personales (`/tasks`):** Nueva interfaz que agrupa las tareas asignadas al usuario logueado. Permite marcar tareas "check" en tiempo real e interactuar con tareas "entregable" mediante un botón táctil de gran área de contacto de 48px ("Ir al Proyecto") para navegación móvil fluida.

### 2. Gestión RBAC Dinámica en Caliente
- **Plantillas de Permisos por Rol:** Se incorpora la tabla `role_permissions_templates` que almacena de forma persistente qué acciones/permisos corresponden a cada nombre de rol.
- **Triggers Reactivos en Base de Datos:** Dos disparadores en PostgreSQL (`on_role_permissions_template_change` y `on_role_permissions_template_delete`) sincronizan de manera instantánea cualquier cambio en las plantillas hacia las tablas físicas `roles` y `role_permissions`. Esto permite actualizar facultades de usuarios en caliente sin alterar código o requerir despliegues.
- **Editor en Orion Console:** Pestaña "Datos & Sistema" en la consola administrativa para que los administradores editen los arrays de permisos de cada rol de manera reactiva e intuitiva.

### 3. Conectividad CRM y Autocompletado Anti-Errores
- **Combobox Inteligente con Creación Inline:** En el formulario de añadir proyecto se integra un Combobox que busca clientes dinámicamente usando coincidencia parcial (`ilike`). Si el cliente no existe, se ofrece la opción rápida de "Crear nuevo cliente" inline de un solo clic, previniendo errores ortográficos o duplicación.
- **Autocompletado de Ocupaciones Sugeridas:** En la edición de perfiles, un input con `datalist` sugiere ocupaciones ya registradas históricamente en la base de datos (`getUniqueOccupations`), unificando nomenclaturas del personal.
- **Restablecimiento de Contraseñas de Empleados:** Un modal interactivo en el panel administrativo permite cambiar la clave de acceso de cualquier empleado de manera directa.

### 4. Seguridad de Endpoints Administrativos
- **Blindaje mediante Bearer Tokens:** Tanto `/api/admin/create-employee` como `/api/admin/reset-password` ahora exigen la cabecera `Authorization: Bearer <token>`.
- **Validación en Backend:** El servidor valida la firma y vigencia del token de sesión de Supabase del usuario solicitante, y realiza una consulta robusta para verificar que posea el rol de 'Administrador' antes de invocar los servicios privilegiados de `auth.admin` (como `createUser` o `updateUserById`).

## Cambios Técnicos & Infraestructura
- **Migración SQL Incremental:** Migración `20260610000100_sprint4_tasks.sql` que crea las tablas `global_tasks` y `role_permissions_templates`, implementa los triggers reactivos y define los datos semilla (seed data) iniciales de plantillas y tareas de prueba.
- **Capa de Servicios Extendida:**
  - `src/core/services/tasks/index.ts` creado con `getTasks`, `createTask`, y `updateTaskStatus`.
  - Funciones auxiliares agregadas en clientes y servicios administrativos.
- **Optimización del Build:** Resolución de bails de prerenderizado Next.js envolviendo las dependencias de `useSearchParams` de la barra lateral en `<Suspense>` en todas las páginas de la aplicación.
