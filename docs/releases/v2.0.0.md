# Release v2.0.0 - Consola Orion, Proyectos & Editor RBAC

**Fecha de Lanzamiento:** 2026-06-09  
**Estado:** Completado y Verificado

---

## Resumen del Lanzamiento
La versión v2.0.0 robustece la arquitectura de datos permitiendo la vinculación de proyectos a clientes, introduce la Consola de Administración de alta densidad e implementa un editor de Roles y Permisos (RBAC) interactivo y en caliente para la gestión de usuarios.

---

## Nuevas Características

### 1. Expansión del Modelo de Clientes & Proyectos
- **Metadatos Avanzados en Clientes:** Se añadieron columnas en base de datos para registrar la categoría (`Residencial`, `Comercial`, `Industrial`), el consumo eléctrico promedio mensual (`avg_kwh_consumption`) y las coordenadas exactas de ubicación (`gps_coordinates`).
- **Módulo de Proyectos (Base de Datos):** Creación de la tabla `projects` con relación 1:N vinculada a `clients`. Protegida mediante políticas RLS multi-tenant completas.
- **Expediente del Cliente (`/clients/[id]`):** Vista detallada del perfil del cliente que incluye:
  - Consumo promedio mensual y categoría.
  - Enlace dinámico directo a Google Maps mediante coordenadas GPS.
  - Listado en cuadrícula de todos los proyectos solares activos asociados a dicho cliente.

### 2. Consola de Administración Orion
- **Cuadrícula de Alta Densidad:** Panel administrativo estilizado con widgets de control rápidos para `[CRM]`, `[PRODUCCIÓN]`, `[INVENTARIO]` y `[DATOS]`.

### 3. Panel de Empleados y Editor de Roles en Caliente (`/admin/users`)
- **Lista de Personal:** Visualización de todos los perfiles registrados bajo la compañía actual.
- **Editor RBAC Dialog:** Modal interactivo para cambiar el rol del usuario (Administrador, Ingeniero, Técnico) y alternar permisos específicos de forma directa utilizando toggles reactivos.
  - Integrado utilizando la API `@base-ui/react` de Dialog para evitar errores de tipado de TypeScript.

---

## Cambios Técnicos & Infraestructura
- **Regeneración de Tipos:** Actualización de los esquemas en `src/core/database/types.ts` mediante Supabase CLI.
- **Servicios Relacionales:** Implementación de la función `getClientProfile` con JOIN relacional (`clients` + `projects`) tipado estrictamente en TypeScript.
- **Refactorización de Rutas:** Se adaptó `DashboardShell` en `src/app/page.tsx` y layouts para preservar la sesión y el sidebar durante la navegación entre rutas sin recargar el estado.
